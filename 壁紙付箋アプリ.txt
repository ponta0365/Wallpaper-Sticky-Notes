
設計としては、メモの本体データは共通にして、表示先だけ切り替える形がいい。

```text
メモDB
  ↓
表示エンジン
  ├─ 壁紙レンダラー
  └─ オーバーレイレンダラー
```

つまり、メモそのものはSQLiteで一元管理。
それを「画像として壁紙に焼き込む」のか、「透明ウィンドウとしてデスクトップ前面に出す」のかを選べるようにする。

モード構成はこう。

```text
表示モード
1. 壁紙モード
   付箋を壁紙画像に合成してWindows壁紙として適用

2. オーバーレイモード
   透明ウィンドウをデスクトップ上に表示して付箋を描画

3. ハイブリッドモード
   通常は壁紙、編集中だけオーバーレイ
```

個人的には3番のハイブリッドがかなり強い。

普段は壁紙に焼き込んで軽量表示。
管理画面を開いたときだけオーバーレイ化して、ドラッグ移動、リサイズ、編集、削除を可能にする。
編集終了したら、現在の配置を壁紙に反映して、オーバーレイを閉じる。

この方式なら、

```text
普段：邪魔にならない
編集時：直接操作できる
終了後：壁紙として残る
```

という形になる。

最小MVPはこれでいいと思う。

機能

・グローバルホットキーで即メモ入力
・Enterで保存
・Shift+Enterで改行
・Escでキャンセル
・メモはSQLiteに保存
・表示モードを選択
　・壁紙モード
　・オーバーレイモード
　・ハイブリッドモード
・モニターごとに表示対象を指定
・付箋の色、位置、サイズを保存
・管理画面で一覧、編集、削除、完了化
・壁紙再生成ボタン
・オーバーレイ編集モードボタン

データ構造はこう。

```text
memos
- id
- body
- status
- color
- monitor_id
- x
- y
- width
- height
- z_index
- created_at
- updated_at
- pinned
- priority

settings
- display_mode
- base_wallpaper_path
- output_wallpaper_dir
- hotkey
- default_monitor_id
- default_note_color
- auto_apply_wallpaper
```

画面構成はこう。

```text
1. クイック入力小窓
2. メモ管理画面
3. オーバーレイ編集画面
4. 設定画面
```

クイック入力小窓はかなりシンプルでいい。

```text
[ メモを入力...                 ]
Enter: 保存 / Shift+Enter: 改行 / Esc: 閉じる
```

オーバーレイ側はこう。

```text
通常表示モード
・付箋を表示
・クリック透過も選べる
・邪魔にならない

編集モード
・付箋をドラッグ移動
・ダブルクリックで編集
・右クリックで色変更、削除、完了
・角をドラッグしてサイズ変更
```

重要なのは、オーバーレイには2種類の状態を持たせること。

```text
閲覧モード：クリック透過ON
編集モード：クリック透過OFF
```

閲覧モードでは、デスクトップ操作の邪魔をしない。
編集モードにしたときだけ付箋を触れる。

この設計はかなり実用的。

技術的には、Pythonならこういう構成がよさそう。

```text
GUI：PySide6
DB：SQLite
画像生成：Pillow
ホットキー：keyboard または pynput
Windows壁紙設定：ctypes / pywin32
オーバーレイ：PySide6 の透過ウィンドウ
```

注意点はここ。

```text
・日本語IME入力中のEnter確定と保存処理の衝突
・マルチモニターの座標管理
・DPIスケーリングによる位置ズレ
・オーバーレイのクリック透過制御
・壁紙変更時の元画像バックアップ
・Windows再起動後の自動復元
```

方向性としてはかなりいい。
このアプリは「付箋アプリ」じゃなくて、「デスクトップを記憶ボード化するアプリ」になる。

完成形の核はこれ。

```text
思いついた瞬間に書く
↓
勝手にデスクトップへ貼られる
↓
必要な時だけ直接並べ替える
↓
普段は壁紙として静かに残る
```

追記
付箋に時刻ボタンをつけてそれを押すと設定した時刻つきメモをOSの通知に飛ばして知らせる
